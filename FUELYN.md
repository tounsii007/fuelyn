## 1. Brand identity

### 1.1 Name

`Fuelyn` — `fuel` + `-yn` (a soft, modern suffix borrowed from companies like *Linear*, *Notion*, *Stripe*). Pronounced *FYOO-lin*.

### 1.2 Logomark

A fuel-drop silhouette with a lightning-shaped "F" cut from the inside, symbolising **intelligence inside fuel data**.

```
   ╱╲
  ╱  ╲
 ╱ ⚡F ╲     ← drop outline + lightning F glyph
╱______╲
```

Living implementation in [`apps/web/src/components/layout/AppShell.tsx`](apps/web/src/components/layout/AppShell.tsx) (the `Brand` component) and [`apps/web/src/components/splash/SplashScreen.tsx`](apps/web/src/components/splash/SplashScreen.tsx).

### 1.3 Color system (OKLCH)

| Role | Token | Hex-ish |
|---|---|---|
| **Brand 500** primary | `--color-brand-500` | electric blue, `oklch(0.62 0.24 245)` |
| **Brand 600** action | `--color-brand-600` | deeper blue (CTAs) |
| **Cyan 400** neon halo | `--color-cyan-400` | bright cyan, used for *live* states + sparkles |
| **Violet 500** AI moments | `--color-violet-500` | premium violet — only for AI-emitted UI |
| **Accent 500** savings | `--color-accent-500` | electric green for "save €" |
| **Background dark** | `--color-bg` | deep midnight navy `oklch(0.13 0.025 252)` |
| **Surface dark** | `--color-surface` | `oklch(0.20 0.030 252)` |

**Rule:** violet appears **only** when the UI was generated by AI (suggestions, predictions). It's the user's signal that "this came from intelligence, not from a static rule."

### 1.4 Glow scale

```css
--shadow-glow-brand:  brand-500 ring + halo  /* primary CTAs, hero card */
--shadow-glow-cyan:   cyan-500  ring + halo  /* live indicators */
--shadow-glow-violet: violet    ring + halo  /* AI elements */
--shadow-glow-success: emerald  ring + halo  /* "you saved" */
--shadow-cinematic: long, soft, slightly bluish  /* hero cards */
```

### 1.5 Typography

| Use | Family | Weight | Size |
|---|---|---|---|
| Display | Inter Variable | 700 | 48 px |
| H1 | Inter Variable | 700 | 32 px |
| H2 | Inter Variable | 600 | 24 px |
| Body | Inter Variable | 400-500 | 14-16 px |
| Code | JetBrains Mono | 400 | 13 px |
| Mono numbers (prices, distances) | Inter `tabular-nums` | 600-700 | varies |

**Letter-spacing:** tight on display (`-0.02em`), uppercase eyebrows go to `0.18em`.

### 1.6 Motion language

```css
--ease-snap:    cubic-bezier(0.32, 0.72, 0, 1);  /* primary */
--ease-spring:  cubic-bezier(0.16, 1, 0.3, 1);   /* sheets, cards */
--ease-soft:    cubic-bezier(0.4, 0, 0.2, 1);    /* fades */
--duration-fast:    150ms;
--duration-default: 250ms;
--duration-slow:    400ms;
```

**Principles:** Every motion has a purpose (state change, attention, feedback). Reduce motion strictly when `prefers-reduced-motion: reduce`. No looping animations longer than 22 s. Microinteractions max 150 ms.

## 4. AI feature catalog

| Feature | What | Backend | UX surface |
|---|---|---|---|
| **Price-drop prediction** | "Prices fall in 2 h — save 6 ct/L" | `ai-service` heuristic + EWMA + Bayesian prior | `PricePredictionCard` |
| **Smart refuel timing** | Best window today/this week | `LocalHeuristicFallback` `bestTimePrediction` | `FuelAdvisor` (live) |
| **Effective price** | Price + drive cost amortised | `effectivePrice()` in heuristic | `BestDealCard.effectiveBest` |
| **Confidence breakdown** | per-signal contributions | `AIAdvisorResponse.breakdown` | inline tooltip in `FuelAdvisor` |
| **Anomaly detection** | Unusual cheap or spike | `BrandBaseline.cheapestBrandZ` + threshold | `AnomalyBadge` on map marker |
| **Driving consumption insights** | Monthly L/100 km, € spent | `ConsumptionTracker` | `/fuel-log` |
| **Year-in-review story** | "Wrapped" personal recap | `wrapped/compute.ts` | `/wrapped` story player |
| **AI chat** | "Wo soll ich heute tanken?" | OpenAI / Ollama enrichment backend | `AIChatDrawer` (TODO) |
| **Geo-fenced alerts** | Crossing a fence triggers a price check | `GeoFenceMount` + watcher | `/alerts` |
| **Personalised loyalty** | Apply Aral Comfort / ADAC to effective price | `AffiliateRebate` | `BestDealCard` (TODO) |

**Architecture note:** Every AI output sits behind a deterministic heuristic baseline. Order: cache → heuristic → Ollama (local) → OpenAI (premium). Hallucinations can never flip `action`/`savings`/`bestStation` — only the prose fields. See [PR commit `b8…`](#) for the tiered pipeline.

## 5. Map experience

### 5.1 Default style

CartoDB **Dark Matter** (already in the stack) is the default in dark mode. Adds a clinical, neutral surface that lets glow markers pop.

### 5.2 Marker grammar

| Type | Visual | Reason |
|---|---|---|
| User location | blue dot + pulsing halo | familiar (Apple/Google) |
| Top deal | violet→cyan glow ring + drop pin | brand emphasis, attention |
| Reachable station | brand-blue circle, filled | safe, neutral |
| Tight reachability | amber outline | warning |
| Unreachable | gray, lower opacity | de-emphasised |
| Charging | cyan lightning bolt | EV mode |
| Hydrogen | violet H₂ glyph | rare → distinctive |
| Favorite | white heart overlay |  user emphasis |

### 5.3 Interactions

- **Hover preview** — dark glass card with name + best price, anchored to marker, 100 ms fade.
- **Click** — bottom-sheet on mobile, side-panel update on desktop.
- **Smart zoom** — auto-fit to result bounds + 10 % padding on every search.
- **Cluster expansion** — spring animation, 250 ms.
- **Route preview** — animated polyline (dash-array offset), 1500 ms ease.

### 5.4 Heatmap (pending)

Aggregate the last 24 h of price snapshots per geohash bucket (precision 6, ~1.2 km). Render as a Leaflet `heatmap.js` overlay; toggle in the top-bar's "Heat" button. Color stops: cheap → green, average → amber, expensive → red, with very low alpha to keep markers legible.

## 6. Animation strategy

### 6.1 Microinteractions

| Element | Trigger | Effect | Duration |
|---|---|---|---|
| Buttons | hover | `scale 1.02` + brand-glow ring | 150 ms |
| Buttons | active | `scale 0.98` | 80 ms |
| Cards | hover | shadow grows, border tints brand | 250 ms |
| Toggle | click | thumb slides + bg color tween | 200 ms |
| Popover | open | spring scale-in + fade | 250 ms |
| Splash | mount | letter-stagger + pulsing ring | 2.4 s |
| BestDealCard | enter | fade-up 8 px + cinematic shadow | 350 ms |

### 6.2 Tools

Currently CSS-only (no `framer-motion`). Pure CSS keyframes + `transition-*` Tailwind utilities cover 95 % of cases at zero runtime cost. Reach for `framer-motion` when we need:
- gesture-driven sheet drags (mobile bottom sheets)
- shared-layout transitions (compare → detail)
- coordinated multi-element choreography (Wrapped)

### 6.3 Reduce-motion

`@media (prefers-reduced-motion: reduce)` already shrinks all durations to 0.01 ms in `tokens.css`. Animations that *convey state* (price update flash, live dot pulse) keep a tiny pulse so the change isn't invisible.

## 7. Performance

### 7.1 Targets

| Metric | Target | Notes |
|---|---|---|
| First Contentful Paint | < 1.5 s | Splash hides cold-cache jank |
| LCP | < 2.5 s | Map tiles preconnect already set |
| Bundle (web entry) | < 250 kB gzipped | Next.js standalone build |
| Map tile cost | < 20 kB / pan | dark Matter at zoom 12-14 |
| Re-render budget | < 16 ms / frame | Zustand selectors, memoised lists |
| Cache hit rate (advisor) | > 80 % | Caffeine 15-min, 0.01° geo-bucket |

### 7.2 Tactics

- **Selectors over context:** `useAppStore((s) => s.filter.fuelType)` — only the consumer of `fuelType` re-renders.
- **Virtualised station list:** for >50 stations, switch to a windowed list (TanStack Virtual).
- **Lazy-load the map:** `next/dynamic` with `ssr: false` (already done).
- **Streaming SSE for prices:** Kafka → SSE Bridge → `usePriceStream` hook (already implemented).
- **Spring Boot warmup:** `start_period: 60s` health-check window, JVM CDS where possible.

## 11. Anti-patterns (do NOT do)

- ❌ Don't use Comic Sans, gradients with > 3 stops, drop shadows wider than 32 px (cinematic only).
- ❌ Don't sprinkle violet outside AI moments — it dilutes the signal.
- ❌ Don't auto-play sound or vibrate.
- ❌ Don't show hardcoded prices in mocks or marketing — always real or labeled `Demo`.
- ❌ Don't hide live errors. If Tankerkönig is down, show it; the heuristic still has 8 signals to lean on.
- ❌ Don't chain `setTimeout` for animation choreography. Use CSS keyframes + `animation-delay`.

## 12. Decision log (key choices we made)

| Decision | Why |
|---|---|
| OKLCH over HSL | perceptually-uniform; same chroma feels equally vibrant across hues |
| `tankpilot.localhost` over `tankpilot.de` | RFC 6761 reserves `*.localhost` for loopback; `tankpilot.de` is a real third-party domain |
| Custom `PopoverSelect` over native `<select>` | native `<option>` chrome is OS-painted, can't be styled (was illegible on dark glass) |
| Tiered AI (heuristic → Ollama → OpenAI) | guarantees correctness even at zero spend; LLM only enriches narrative |
| Kafka via Redpanda over Apache Kafka | ~256 MB RAM idle vs ~1 GB; same wire protocol |
| CSS variables for mesh backgrounds | eliminates selector-specificity ambiguity; one rule consumes the vars |
| Reverse-geocode at zoom 18 | `zoom=14` (default) returns no road/house_number — only PLZ + city |
| `enableHighAccuracy: true` for explicit location clicks | coarse Wi-Fi triangulation gives 100+ m error which makes street-level reverse impossible |
